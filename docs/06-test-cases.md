# Тестовые сценарии

| ID | Сценарий | Ожидаемый результат |
|---|---|---|
| TC-01 | Полученный товар возвращают через 5 дней | Заявка создаётся и получает `APPROVED` |
| TC-02 | Товар возвращают через 15 дней | Заявка получает `REJECTED`, указана причина |
| TC-03 | Заказ принадлежит другому покупателю | Ошибка доступа, заявка не создаётся |
| TC-04 | Запрошено 2 штуки, но получена 1 | Ошибка количества |
| TC-05 | По позиции уже есть активный возврат | Новая заявка не создаётся |
| TC-06 | Покупатель отменяет заявку до передачи | Статус меняется на `CANCELLED` |
| TC-07 | Покупатель отменяет товар в пути | Ошибка `409`, статус не меняется |
| TC-08 | Склад принял весь товар | Рассчитана полная сумма возврата |
| TC-09 | Склад принял 1 товар из 2 | Рассчитана сумма за 1 товар |
| TC-10 | Склад отказал по всем позициям | `COMPLETED_WITHOUT_REFUND` |
| TC-11 | Payment Service вернул успешный результат | Заявка получает `COMPLETED` |
| TC-12 | Payment Service вернул ошибку | Заявка остаётся `REFUND_IN_PROGRESS`, ошибка записывается |
| TC-13 | В запросе нет обязательного `orderId` | Ошибка `400` с названием поля |
| TC-14 | Запрашивается неизвестная заявка | Ошибка `404` |

## Пример критерия приёмки

```gherkin
Given покупатель получил товар 5 дней назад
And по товару нет активного возврата
When покупатель создаёт заявку на возврат одной единицы
Then система создаёт заявку
And заявка получает статус APPROVED
And покупателю показывается номер заявки
```
