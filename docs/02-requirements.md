# Требования и бизнес-правила

## Функциональные требования

| ID | Требование |
|---|---|
| FR-01 | Покупатель может создать заявку на возврат по своему заказу. |
| FR-02 | В одной заявке можно выбрать несколько позиций одного заказа. |
| FR-03 | Для каждой позиции указываются количество, причина и комментарий. |
| FR-04 | Система проверяет заказ и дату получения через Order Service. |
| FR-05 | Система проверяет срок возврата и доступное количество товара. |
| FR-06 | После проверки система одобряет или отклоняет заявку. |
| FR-07 | При отказе система сохраняет и показывает причину. |
| FR-08 | Покупатель выбирает передачу курьеру или в пункт приёма. |
| FR-09 | Система получает статусы обратной доставки. |
| FR-10 | Покупатель может посмотреть текущий статус заявки. |
| FR-11 | Покупатель может отменить заявку до передачи товара. |
| FR-12 | Склад может указать принятое и отклонённое количество. |
| FR-13 | Система рассчитывает сумму возврата по принятому количеству. |
| FR-14 | Система отправляет запрос на возврат денег в Payment Service. |
| FR-15 | После ответа Payment Service заявка получает конечный статус. |
| FR-16 | Изменения статуса сохраняются в истории. |
| FR-17 | После важных этапов покупателю отправляется уведомление. |

## Бизнес-правила

| ID | Правило |
|---|---|
| BR-01 | Вернуть можно только товар из заказа текущего покупателя. |
| BR-02 | Позиция заказа должна быть получена. |
| BR-03 | Заявка создаётся не позднее 14 дней после получения товара. |
| BR-04 | Количество должно быть больше нуля и не больше полученного количества. |
| BR-05 | Нельзя создать второй активный возврат для той же позиции. |
| BR-06 | Некоторые категории нельзя вернуть без дефекта. Список приходит из Order Service вместе с данными позиции. |
| BR-07 | После передачи товара покупатель уже не может отменить заявку. |
| BR-08 | Склад должен указать причину, если товар не принят. |
| BR-09 | Сумма возврата равна цене единицы, умноженной на принятое количество. |
| BR-10 | Сумма возврата не может быть больше оплаченной суммы позиции. |

## Нефункциональные требования

| ID | Требование |
|---|---|
| NFR-01 | Ответ на просмотр заявки должен приходить не дольше 2 секунд при нормальной работе систем. |
| NFR-02 | API использует HTTPS и JSON. |
| NFR-03 | Доступ к заявке есть только у её владельца и сотрудников с нужной ролью. |
| NFR-04 | Для каждого изменения статуса сохраняются время и источник. |
| NFR-05 | Ошибки внешних систем записываются в журнал. |
| NFR-06 | Дата и время передаются в UTC в формате ISO 8601. |
