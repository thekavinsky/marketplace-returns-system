# API и интеграции

Полное описание методов находится в [OpenAPI](../api/openapi.yaml).

## Методы ReturnFlow

| Метод | URL | Назначение |
|---|---|---|
| POST | `/returns` | Создать заявку |
| GET | `/returns/{returnId}` | Получить заявку и статус |
| POST | `/returns/{returnId}/cancel` | Отменить заявку |
| POST | `/returns/{returnId}/delivery-status` | Получить новый статус доставки |
| POST | `/returns/{returnId}/inspection` | Сохранить проверку склада |
| POST | `/returns/{returnId}/payment-result` | Получить результат возврата денег |

Первые три метода используются приложением покупателя. Остальные вызываются внутренними системами.

## Взаимодействие с системами

### Order Service

При создании заявки ReturnFlow запрашивает заказ по `orderId`. Нужны владелец, статус позиции, дата получения, цена и количество.

### Delivery Service

После одобрения заявки ReturnFlow создаёт обратную доставку. Delivery Service сообщает, когда товар передан и когда он приехал на склад.

### Payment Service

После складской проверки ReturnFlow передаёт идентификатор заявки и сумму. Payment Service сообщает успешный или неуспешный результат.

### Notification Service

ReturnFlow отправляет команду на уведомление после создания заявки, получения товара и возврата денег.

## Пример ошибки

```json
{
  "code": "RETURN_PERIOD_EXPIRED",
  "message": "Срок возврата товара закончился"
}
```

Основные HTTP-коды:

- `400` — ошибка в данных запроса;
- `401` — пользователь не авторизован;
- `403` — нет доступа к заказу или заявке;
- `404` — заявка не найдена;
- `409` — действие нельзя выполнить в текущем статусе;
- `500` — внутренняя ошибка;
- `503` — временно недоступна внешняя система.
