# ReturnFlow

Учебный проект по системному анализу. Я выбрал процесс возврата товара в маркетплейсе, потому что в нём участвует сразу несколько систем: заказы, доставка, склад и оплата.

Цель проекта — описать, как покупатель создаёт заявку на возврат, передаёт товар, а после проверки получает деньги.

## Что есть в проекте

- описание задачи и границ системы;
- функциональные и нефункциональные требования;
- бизнес-правила;
- Use Case основного сценария;
- диаграммы процесса, состояний и взаимодействия систем;
- простой REST API в формате OpenAPI;
- модель базы данных и SQL-скрипт;
- несколько тестовых сценариев;
- открытые вопросы, которые нужно уточнять у заказчика.

## Основной процесс

1. Покупатель выбирает товар из полученного заказа.
2. Система проверяет, можно ли его вернуть.
3. Покупатель передаёт товар курьеру или в пункт приёма.
4. Склад проверяет состояние товара.
5. Если возврат принят, платёжная система возвращает деньги.

## Границы проекта

В проекте рассматривается возврат уже полученного товара. Обмен товара, споры с продавцом, международные заказы и пользовательский интерфейс не рассматриваются.

Это учебная модель. В настоящем проекте сроки возврата, список ограниченных категорий и порядок проверки нужно согласовать с бизнесом и юристами.
